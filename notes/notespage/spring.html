Spring

我为什么选择会去先学习spring框架？

因为spring框架虽然时间比较长，但是还没有过时，现在仍有许多人仍在使用spirng框架，且很多后端工程师学习java后选择的框架就是spring框架，目前最通用的解决方案就算SSH。所以从这里我认为spring框架可能更容易学习，更容易入手，相关的学习资料也更多，如果有相关问题也可以问更多的人，所以我选择了spring为我自主学习的第一个框架。

因为我原来都是做前端，对于后端也只是有限的接触了flask框架，而python并不是主要做后端的，所以我称spring为我接触的第一个后端框架而不是称flask是。

flask当然是很好的后端框架，但是为了顺从主流我还是选择将flask放到一边去。

Spring 被称为 J2EE 的春天，是一个开源的轻量级的 Java 开发框架， 具有控制反转（IoC）和面向切面（AOP）两大核心。Java Spring 框架通过声明式方式灵活地进行事务的管理，提高开发效率和质量。

Spring 框架不仅限于服务器端的开发。从简单性、可测试性和松耦合的角度而言，任何 Java 应用都可以从 Spring 中受益。Spring 框架还是一个超级粘合平台，除了自己提供功能外，还提供粘合其他技术和框架的能力。
